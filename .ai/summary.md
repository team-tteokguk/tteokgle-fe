## 🤖 AI Change Summary for PR #30

### 개요
이번 변경 사항은 인증 관련 기능 개선, 아이템 배치 로직 및 오류 처리 개선, CSS 스타일 추가, 새로운 로그인 및 닉네임 설정 페이지 구현, 라우터 설정 업데이트, Axios 인스턴스 수정, 그리고 새로운 컴포넌트 추가를 포함합니다. 주요 목적은 사용자 인증 흐름을 개선하고, 코드의 가독성과 유지보수성을 높이며, UI의 일관성을 유지하는 것입니다.

### 주요 변경 사항
1. **인증 관련 코드 변경**:
   - `ProtectedRoute` 컴포넌트에서 사용자 인증 상태를 `user`에서 `isAuthenticated`로 변경.
   - `useAuth` 훅에서 `clearStore`를 `clearAuth`로 변경하고, `setAccessToken`을 추가하여 인증 상태 관리 개선.
   - Axios 요청 인터셉터에 액세스 토큰 추가 및 응답 인터셉터에서 401 오류 발생 시 리프레시 토큰 요청 로직 추가.

2. **아이템 배치 로직 개선**:
   - `onSuccess` 콜백에서 인자로 `itemId`를 받아 쿼리 무효화 로직을 개선.
   - `AddItemModal`에서 아이템 클릭 시 기본 요청 본문을 정의하여 `placementItem` 호출 시 전달.

3. **새로운 페이지 및 컴포넌트 추가**:
   - 구글 로그인 콜백 처리 페이지(`GoogleCallback`), 로그인 페이지(`Login`), 닉네임 설정 페이지(`SetupNickname`) 추가.
   - `BouncingEmojis` 컴포넌트 추가, 애니메이션 효과를 가진 이모지 표시.

4. **CSS 스타일 추가**:
   - 비활성화된 버튼을 위한 새로운 그라디언트 스타일(`--grad-disabled`) 추가 및 소셜 로그인 버튼 스타일 정의.

5. **라우터 설정 변경**:
   - 로그인 경로(`/login`) 및 구글 인증 콜백 경로(`/auth/google/callback`) 추가, 기존의 메인 페이지는 제거됨.

6. **테스트 코드 개선**:
   - `useMyItem` 훅의 테스트에서 데이터 존재 여부를 확인하는 조건문 추가.
   - `useUpdateItemPlacement` 훅의 `mutationFn`에서 인자를 객체로 묶어 전달하도록 수정.

### 위험/영향
- 인증 흐름 변경으로 인해 사용자 인증 오류가 발생할 수 있으며, 충분한 테스트가 필요합니다.
- 새로운 페이지 추가로 기존 라우팅 로직에 영향을 줄 수 있으며, 경로 관리에 주의가 필요합니다.
- CSS 스타일 변경으로 UI의 일관성이 영향을 받을 수 있습니다.
- `peer` 속성 제거로 인해 일부 패키지의 의존성 관리에 영향을 줄 수 있습니다.

### 테스트/검증
- 변경된 인증 흐름과 아이템 배치 기능에 대한 단위 테스트 및 통합 테스트를 수행해야 합니다.
- 새로운 로그인 및 닉네임 설정 페이지의 기능을 검증하기 위해 각 페이지의 UI 및 API 호출을 테스트해야 합니다.
- Axios 요청 및 응답 인터셉터의 동작 확인, 특히 401 오류 처리 및 리프레시 토큰 요청 테스트.
- CSS 스타일이 적용된 버튼의 시각적 요소를 확인하여 디자인 요구 사항을 충족하는지 검증해야 합니다.

### 후속 조치
- 인증 관련 기능에 대한 사용자 피드백을 수집하고, 필요시 추가적인 수정 작업을 진행해야 합니다.
- 아이템 배치 기능의 오류 처리 로직을 모니터링하여 추가적인 버그 수정이 필요한지 확인해야 합니다.
- CSS 스타일이 전체 애플리케이션에 일관되게 적용되도록 검토하고, 필요한 경우 추가적인 스타일 조정을 진행해야 합니다.
- 변경된 사항 및 새로운 기능에 대한 문서화 작업을 진행해야 합니다.
