## 🤖 AI Change Summary for PR #33

### 개요
이번 변경 사항은 상점 관련 API와 상태 관리 로직을 개선하고, 친구 찾기 기능 및 관련 UI를 추가하여 사용자 경험을 향상시키기 위한 것입니다. 주요 변경 사항으로는 즐겨찾기 기능, 상점 검색 기능, 그리고 친구 찾기 기능이 포함됩니다.

### 주요 변경 사항
1. **API 수정**:
   - 즐겨찾기 추가 및 해제 API 경로 변경: `/store/{storeId}/subscription`에서 `/stores/{storeId}/subscribe`로 수정.
   - 새로운 API 메서드 추가:
     - `searchStores`: 상점 검색 기능.
     - `getMyFavoriteStores`: 사용자의 즐겨찾기 목록 조회 기능.

2. **상태 관리 키 추가**:
   - `storeKeys`에 즐겨찾기 목록과 검색 목록 관련 키 추가.
   - 친구 찾기 기능을 위한 상태 관리: `activeTab`, `hiddenStoreIds`, `optimisticFavorites`, `searchInput`, `keyword` 추가.

3. **훅 추가**:
   - `useSearchStores`: 상점 검색을 위한 무한 스크롤 훅 추가.
   - `useGetFavoriteStores`: 사용자의 즐겨찾기 상점 목록을 가져오는 훅 추가.
   - `useToggleStoreFavorite`: 즐겨찾기 상태를 변경하는 훅 추가.

4. **타입 정의 추가**:
   - `StoreFavoritesRequest`, `StoreListSliceResponse`, `StoreSearchRequest` 인터페이스 추가.

5. **스타일 및 아이콘 추가**:
   - 새로운 CSS 클래스 `.find-buddy-button-base` 추가.
   - 검색 아이콘 및 즐겨찾기 관련 아이콘(활성화 및 비활성화 상태) 추가.

6. **새로운 페이지 및 컴포넌트 추가**:
   - `FindFriends.tsx` 페이지 추가, 상점 검색 및 즐겨찾기 기능을 포함.
   - `FindFriends` 컴포넌트를 라우터에 추가하여 `/find-buddy` 경로에서 접근 가능하도록 설정.

### 위험/영향
- API 경로 변경으로 인해 기존 클라이언트 코드에서 오류가 발생할 수 있으며, 클라이언트 코드의 업데이트가 필요합니다.
- 새로운 훅과 API 메서드가 추가됨에 따라 기존 코드와의 호환성 문제를 검토해야 합니다.
- 새로운 상태 관리 로직이 추가되어 복잡성이 증가할 수 있으며, 잘못된 상태 업데이트가 발생할 경우 UI에 영향을 미칠 수 있습니다.
- 새로운 페이지 추가로 인해 UI/UX 테스트가 필요하며, 성능 저하가 발생할 수 있는 위험이 존재합니다.

### 테스트/검증
- API 경로 변경에 따른 기존 기능의 정상 작동 여부를 확인하기 위한 단위 테스트 및 통합 테스트 필요.
- 친구 찾기 기능이 정상적으로 작동하는지, 검색 및 즐겨찾기 기능이 의도한 대로 작동하는지 확인해야 합니다.
- 다양한 화면 크기에서 UI가 올바르게 표시되는지 검증하고, 상태 업데이트가 올바르게 이루어지는지 확인하기 위한 단위 테스트 필요.

### 후속 조치
- 클라이언트 코드에서 변경된 API 경로를 반영하도록 업데이트.
- 초기 사용자 피드백을 통해 발견된 버그를 수정하고, 새로운 기능에 대한 사용자 피드백 수집 및 개선 사항 반영.
- 성능 모니터링을 통해 필요 시 최적화 작업 진행 및 추가적인 테스트 케이스 작성 및 코드 리뷰를 통해 품질 보증.
